## 🎨 Theme & Layout Rules (CRITICAL)
Do **NOT** write arbitrary Tailwind color classes (e.g., `bg-[#faf9f6]` or `dark:bg-stone-950`) inside component markup. You must exclusively use the semantic abstraction design tokens defined inside `src/index.css` via Tailwind v4’s `@theme` directive.

### Permitted Class Tokens Matrix:
* **Backgrounds:** `bg-canvas`, `bg-surface`, `bg-surface-muted`, `bg-brand-muted`
* **Text Controls:** `text-txt-main`, `text-txt-muted`, `text-brand`
* **Borders/Lines:** `border-line`
* **Shadows:** `shadow-classy`
* **Typography Elements:** `font-serif` (for editorial headings), `font-sans` (for layout body and navigation UI).

*Note: Dark/Light transitions are natively handled under the hood by these tokens when the `.dark` class is attached to the root document wrapper element.*
